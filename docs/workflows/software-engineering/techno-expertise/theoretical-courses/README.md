# Theoretical Courses

Academic and mathematical concepts for deep understanding.

---

## Purpose

Understand **fundamental concepts** that underlie all software:

- Why algorithms work
- Mathematical foundations
- Formal methods
- Theory of computation

---

## Difference from Other Types

| Aspect | Techno Course | Need-Based Guide | Theoretical Course |
|--------|---------------|------------------|-------------------|
| Focus | Technology | Problem | Concept |
| Examples | Code | Cross-tech | Math, diagrams |
| "Why" | No | Sometimes | Yes |
| Audience | All devs | Problem solvers | Senior/Architects |

---

## Example Structure

```markdown
Graph Theory
├── README.md              # Overview, why it matters
├── fundamentals/
│   ├── definitions.md    # Vertices, edges, directed
│   ├── representations.md # Matrix, list
│   └── traversals.md     # BFS, DFS
├── algorithms/
│   ├── shortest-path.md  # Dijkstra, Bellman-Ford
│   ├── spanning-tree.md  # Prim, Kruskal
│   └── matching.md       # Bipartite matching
├── applications/
│   ├── social-networks.md # Recommendations
│   ├── routing.md        # GPS, networks
│   └── dependency-resolution.md # Package managers
└── examples/
    ├── implementation.ts # Code in practice
    └── visualizations/   # Diagrams
```

## Topics to Cover

### Discrete Mathematics

- **Graph Theory** - Paths, cycles, algorithms
- **Automata** - State machines, regex, parsing
- **Formal Languages** - Grammars, BNF

### Programming Theory

- **Compilers** - Lexing, parsing, optimization
- **Type Theory** - Type systems, lambda calculus
- **Category Theory** - Monads, functors (advanced)

### Statistics & Probability

- **Probability** - Distributions, Bayes theorem
- **Statistics** - Inference, hypothesis testing
- **Stochastic Processes** - Markov chains

---

## For Developers

Why learn theory?

1. **Better Algorithms** - Understand why, not just how
2. **System Design** - Foundation for architecture
3. **Problem Solving** - Tools for complex challenges
4. **Career Growth** - Senior/Staff/Principal level

---

## When to Use

| Situation | Use |
|-----------|-----|
| Need dependency resolution | → Graph Theory |
| Need state machines | → Automata Theory |
| Need to understand types | → Type Theory |
| Need ML fundamentals | → Probability & Stats |

---

## Complexity Level

Theoretical courses are more challenging to create:

| Challenge | Impact |
|-----------|--------|
| Mathematical rigor | Must be accurate |
| Accessibility | Hard to explain simply |
| Examples | Need practical applications |
| Validation | Harder to verify |
