## The Issue Lifecycle

```
ISSUE CREATED → TRIAGE → READY → IN-PROGRESS → REVIEW → DONE
     │           │       │         │           │        │
     ▼           ▼       ▼         ▼           ▼        ▼
  New issue  Evaluate Prioritize  Working   Code       Merged
                           by priority     complete
```

---

## Key Principles

1. **Every issue needs labels** - Type, Status, Priority, Effort
2. **Every issue needs acceptance criteria** - How do we know it's done?
3. **Priority drives work order** - Critical first, then high, etc.
4. **Agents use labels** - To know what to work on next
